# Network Access Control Lists (NACLs)

## Overview

A custom Network ACL was configured for the private application subnets to provide subnet-level traffic filtering in addition to Security Groups.

## Inbound Rules

| Rule | Protocol | Port | Source | Action |
|------|----------|------|--------|--------|
|100|HTTP|80|10.0.0.0/16|Allow|
|110|SSH|22|10.0.1.0/24|Allow|
|120|TCP|1024–65535|0.0.0.0/0|Allow|
|*|All Traffic|All|0.0.0.0/0|Deny|

## Outbound Rules

| Rule | Protocol | Port | Destination | Action |
|------|----------|------|-------------|--------|
|100|HTTP|80|0.0.0.0/0|Allow|
|110|HTTPS|443|0.0.0.0/0|Allow|
|120|TCP|1024–65535|0.0.0.0/0|Allow|
|*|All Traffic|All|0.0.0.0/0|Deny|

## Purpose

The custom NACL provides an additional layer of security by filtering traffic at the subnet level while Security Groups continue to protect individual EC2 instances.
