# EC2 Instances

## Overview

Amazon Elastic Compute Cloud (Amazon EC2) provides scalable virtual servers in the AWS Cloud. In this project, EC2 instances were used to host the Bastion Host, Apache web servers, and the PostgreSQL database server.

Each instance was assigned a specific role within the three-tier architecture to ensure security, scalability, and efficient resource utilization.

---

## Objectives

The EC2 instances were deployed to:

- Host the web application.
- Provide secure administrative access.
- Host the PostgreSQL database.
- Separate application and database layers.
- Support a secure three-tier architecture.

---

## EC2 Instance Configuration

| Instance | Purpose | Subnet |
|----------|---------|--------|
| Bastion Host | Secure SSH Access | Public Subnet |
| App Server 1 | Apache Web Server | Private App Subnet |
| App Server 2 | Apache Web Server | Private App Subnet |
| PostgreSQL Server | Database Server | Private Database Subnet |

---

## Bastion Host

The Bastion Host acts as a secure entry point for administrators to access resources located in private subnets.

### Responsibilities

- SSH access to private EC2 instances
- Secure administration
- Prevent direct internet access to backend servers

---

## Application Servers

Two EC2 instances were configured as Apache Web Servers.

### Responsibilities

- Host the web application
- Process incoming requests
- Receive traffic from the Application Load Balancer
- Connect securely to the PostgreSQL database

### Software Installed

- Apache HTTP Server
- OpenSSL
- SSL Certificates
- Website Files

---

## Backend HTTPS

To secure communication between the Application Load Balancer and the application servers, Apache was configured with a self-signed SSL certificate.

Benefits include:

- Encrypted backend communication
- Secure data transmission
- Successful HTTPS health checks from the Application Load Balancer

---

## PostgreSQL Database Server

A dedicated EC2 instance was configured as the PostgreSQL database server.

### Responsibilities

- Store application data
- Accept connections only from application servers
- Operate entirely within a private subnet

### Software Installed

- PostgreSQL
- PostgreSQL Client Tools

---

## Communication Flow

```text
Administrator
      │
 SSH (22)
      │
Bastion Host
      │
 SSH (22)
      ▼
Application Servers
      │
 HTTPS (443)
      ▼
Application Load Balancer
      │
 PostgreSQL (5432)
      ▼
Database Server
```

---

## Security Measures

- Bastion Host deployed in a public subnet.
- Application servers deployed in private subnets.
- Database server isolated in a private subnet.
- SSH access restricted through the Bastion Host.
- Backend communication secured using HTTPS.
- Database accessible only from the application servers.

---

## High Availability

The application layer consists of two Apache web servers deployed across different Availability Zones.

Advantages include:

- Improved reliability
- Load distribution
- Fault tolerance
- Increased application availability

---

## Benefits

- Scalable virtual infrastructure
- Secure deployment
- Separation of application and database layers
- Flexible resource management
- High availability support

---

## Outcome

The EC2 instances form the core computing infrastructure of the AWS 3-Tier Web Hosting Architecture. By separating the Bastion Host, application servers, and PostgreSQL database into dedicated instances, the architecture achieves improved security, scalability, and maintainability while following AWS best practices.
